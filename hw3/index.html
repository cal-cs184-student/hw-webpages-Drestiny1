Part 1: Ray Generation and Scene Intersection

For ray generation and rendering, I send rays from the camera to the scene to figure out what the pixels should look like. If the ray hits an object, I use information about the surface to determine the color and if the ray doesn't hit anything, the pixel stays empty.

I also implemented the triangle intersection algorithm, which tests whether a ray intersects a triangle. I used the Möller–Trumbore algorithm, which computes where the ray would hit the triangle and uses weights to determine how close the point is to each corner. If the point is inside the triangle and in front of the camera, the ray hits the triangle.

Part 2: Bounding Volume Hierarchy

To speed up rendering, I used Bounding Volume Hierarchy (BVH), which uses a tree structure to group nearby objects together in boxes. Instead of checking each object one by one, the program first checks the larger boxes, choosing a direction, and then continuing until the boxes are small enough. This makes rendering a lot faster.

Rendering without BVH is O(n). Rendering with BVH is around O(log n) because of the tree structure and faster search. Rendering the bunny image without using BVH takes over 498 seconds. Rendering it with BVH took around 0.53 seconds.

Part 3: Direct Illumination

I implemented direct lighting two different ways. The first is hemisphere sampling and the second is importance sampling.

In Hemisphere Sampling, we estimate incoming light from all directions uniformly over the hemisphere at each point. This method is inefficient because it samples even in directions that don't contribute light.

In Importance Sampling, we focus on directions that contribute light. For each light in the scene, we sample points between the light source and the point. If there is no other object between the light source and the point, then we know that there's light.

Hemisphere Sampling is often produces more noise compared to Importance sampling because it samples many different directions that don't produce light. For Importance Sampling, as the number of light sources increases, shadows become smoother, whereas in hemisphere sampling, it remains noisy even at higher sample counts

Part 4: Global Illumination

I implemented indirect lighting, which simulates how light reflects off an object. When a ray hits an object, I first compute the direct lighting from the light source. Then I randomly choose a new direction for the ray to bounce off using the material's properties. Each bounce contributes less light. I also use a technique called Russian Roulette, which randomly stops some ray bounces early and adjust their contribution so that the final result stays accurate. Overall, this allows the lighting on the rendered image to look more realistic.

Accumulate Bounce = True sums all of the bounces from 0 up to the specified depth. Accumulate Bounce = False only uses the light from the nth bounce.

As you can see in the images above, accumulate bounce = false often results in darker images. It's useful for analyzing the effect of individual bounces. We can see that most of the light come from the first few bounces and they gradually grow darker after the 2nd bounce. The second bounce is especially important because it introduces indirect lighting. The third bounce and bounces after that introduce softer light and further refine the image to make it look more natural and realistic compared to rasterization.

Part 5: Adaptive Sampling

Adaptive Sampling makes rendering more efficient by spending more time rendering noisy pixels and less time on pixels that have already converged. Adaptive sampling monitors how stable a pixel's brightness is when new samples are added. For each batch, I compute the mean and variance to get a 95% confidence interval and once the pixel has converged, I stop sampling. In this way, it saves time by focusing on areas that are harder to render, while stopping early for areas that are simpler.

In the Sampling Rate Images, warming colors like red indicate that the pixels require more time to converge due to higher variance. Cooler colors like blue indicate that the pixels converge quickly such as flat surfaces or evenly lit areas.
